# Visual World

## Prerequisite

- Install of Docker community <https://docs.docker.com/install/>.
- Create a folder owner by systemd-coredump with RXW permissions, dedicated for mongo (eg. `~/data/mongo`).

## Install

Clone Git repo from <https://github.com/jdequatrebarbeszenvalue/visual-world>

## Config

The folder `[visual-world-folder]/conf.sample` contains samples of config files.

It must be renamed from `conf.sample` to `conf` before use.  
A symlink may also be created: `ln -s ./conf.sample/  ./conf`

It may contains:
- `httpd.conf`: config file for front (required);
- `proxy.cfg`: config file for proxy (required);
- `ssl/publickey.pem`: cert for https;
- `ssl/privatekey.pem`: private key for https.

*Nota bene*: public and private key may be generated by `autocert` with the following process:
1. Visual World must be down;
2. launch `get.certificates.sh`;
3. copy `/etc/letsencrypt/live/[your.domain.name]/cert.pem`  
     to `[visual-world-folder]/conf/ssl/publicKey.pem`;
4. copy `/etc/letsencrypt/live/[your.domain.name]/privKey.pem`  
     to `[visual-world-folder]/conf/ssl/privateKey.pem`.

## Launch

Docker service must be up.  
Launch `start.prod.sh`# Test
